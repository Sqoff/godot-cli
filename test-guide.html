STEP 01 / 05

HTTP Server + Token Auth

📋 준비 Godot → Project Settings → Plugins → GodotCLI 활성화
🔍 확인 Output 패널에서 [godot-cli] Server running on port {PORT} · Token: {TOKEN} 복사
T1인증된 ping → 200 OK
$ curl -s -X POST http://localhost:{PORT}/api/ping -H "Authorization: Bearer {TOKEN}"
→ {"success": true, "data": {"pong": true}}
T2토큰 없음 → 401 Unauthorized
$ curl -s -X POST http://localhost:{PORT}/api/ping
→ {"success": false, "error": {"code": "UNAUTHORIZED", "message": "Invalid or missing token"}}
T3없는 엔드포인트 → 404 Not Found
$ curl -s -X POST http://localhost:{PORT}/api/unknown -H "Authorization: Bearer {TOKEN}"
→ {"success": false, "error": {"code": "COMMAND_NOT_FOUND", ...}}

STEP 02 / 05

Instance Management

T1다중 인스턴스 → 각각 별도 파일 생성
$ ls ~/.godot-cli/instances/
→ 12345.json 67890.json (각 인스턴스당 1개, 포트·토큰 다름)
T2인스턴스 파일 구조 — 7개 필드 전체 확인
$ cat ~/.godot-cli/instances/12345.json # Win: %USERPROFILE%\.godot-cli\instances\12345.json
"pid": 12345 · "port": 8090 · "godot_version": "4.4.1"
"project_path": "/path/to/my-game/"
"token": "a1b2c3d4e5f6...(32자 hex)"
"started_at": "2026-05-14T10:30:00" · "last_heartbeat": "2026-05-14T10:30:10"
T3에디터 종료 → 파일 자동 삭제
# Godot 에디터 닫기 후
$ ls ~/.godot-cli/instances/
→ 빈 디렉토리 (파일 없음)
T4하트비트 10초 간격 갱신 확인
# 10초 대기 후 last_heartbeat 값 비교
$ cat ~/.godot-cli/instances/12345.json | python -m json.tool | grep heartbeat
→ 타임스탬프가 이전 값과 달라야 함 (10초 이내 갱신)

STEP 03 / 05

Go CLI Skeleton

🔨 빌드 go build -o godot-cli .
테스트 방법 · 기대 결과
T1status — 에디터 정보 출력
$ ./godot-cli status
→ Project: MyGame · Godot 4.4.1 · Scene: main.tscn
T2--json 플래그 — 구조화 출력
$ ./godot-cli status --json
→ {"project": "MyGame", "godot_version": "4.4.1", ...}
T3좀비 파일 자동 정리
# Godot 비정상 종료 후 인스턴스 파일 남긴 상태
$ ./godot-cli list
→ PID 미존재 파일 자동 삭제 후 활성 목록만 표시
종료 코드 검증
상황종료코드확인 방법
정상 실행0echo $?
커맨드 에러1서버 에러 응답 시
연결 실패2Godot 없이 실행
인증 실패3잘못된 토큰으로 실행
T4연결 없음 → 종료 코드 2
$ ./godot-cli status; echo "exit: $?"
→ no active Godot editor instance found · exit: 2

STEP 04 / 05

MVP Commands (7개)

커맨드 → 테스트 방법 · 기대 결과
커맨드기대 출력확인
godot-cli runGame started에디터 재생 버튼 활성화
godot-cli run --scene res://main.tscnGame started scene=…해당 씬으로 재생
godot-cli stopGame stopped에디터 정지 버튼 활성화
godot-cli sceneScene: res://…tscn에디터 현재 탭과 일치
godot-cli scene set res://other.tscnScene: res://other.tscn에디터 탭이 변경됨
godot-cli exec . get_name"MyScene"씬 루트 이름 반환
godot-cli connectConnected port=…연결 성공 확인
exec 보안 검증
T1exec 기본 비활성화 → EXEC_DISABLED
$ ./godot-cli exec . get_name
→ exec is disabled; enable via Editor > Editor Settings > godot_cli/enable_exec
→ exit code 1
T2enable_exec=true 후 메서드 호출
# Editor Settings → godot_cli/enable_exec = true
$ ./godot-cli exec . get_name --json
→ {"result": "MyScene"} · exit code 0
T3존재하지 않는 노드 → NODE_NOT_FOUND
$ ./godot-cli exec Ghost fire
→ Node not found: Ghost · exit code 1
T4존재하지 않는 메서드 → METHOD_NOT_FOUND
$ ./godot-cli exec . nonexistent_fn
→ Method 'nonexistent_fn' not found · exit code 1

Build & Deploy

빌드 검증 · 테스트 방법
T1로컬 빌드 — 바이너리 생성 확인
$ go mod tidy
$ go build -o godot-cli.exe .
$ ./godot-cli.exe --help
→ connect, exec, list, run, scene, status, stop 커맨드 포함 usage 출력
T2GoReleaser 스냅샷 — 3 플랫폼 크로스 빌드
$ goreleaser build --clean --snapshot
$ ls dist/
→ godot-cli_linux_amd64/
   godot-cli_darwin_arm64/
   godot-cli_darwin_amd64/
   godot-cli_windows_amd64/
T3체크섬 파일 생성 확인
$ goreleaser release --clean --skip=publish
$ cat dist/checksums.txt
→ sha256 ... godot-cli_linux_amd64.tar.gz 등 각 파일별 해시
HTTP API 직접 검증 (curl)
T4토큰 없음 → 401 UNAUTHORIZED
$ curl -s -X POST http://127.0.0.1:8090/api/ping | jq .
→ {"success":false,"error":{"code":"UNAUTHORIZED","message":"Invalid or missing token"}}
T5없는 커맨드 → 404 COMMAND_NOT_FOUND
$ curl -s -X POST http://127.0.0.1:8090/api/nope \
   -H "Authorization: Bearer <TOKEN>" | jq .
→ {"success":false,"error":{"code":"COMMAND_NOT_FOUND",...}}
에러 코드 참조
코드HTTP트리거
UNAUTHORIZED401Bearer 토큰 없음·불일치
COMMAND_NOT_FOUND404없는 /api/ 엔드포인트
INVALID_PARAMS400필수 필드 누락·잘못된 action
EXEC_DISABLED403enable_exec = false 상태
NODE_NOT_FOUND404node_path 노드 없음
METHOD_NOT_FOUND404해당 노드에 메서드 없음
NOT_READY503레지스트리 미초기화 (버그)
